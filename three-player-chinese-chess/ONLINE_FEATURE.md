# 联机在线功能维护与部署说明

本文档记录当前“三人中国象棋”联机在线功能的实现成果，方便后续维护、排查和部署发布。

## 1. 功能概览

当前联机在线为第一版房间码联机：

- 房间码为 5 位数字。
- 玩家在首页选择“联机在线”后，可在卡片右侧输入房间码。
- 房间码留空点击“开始”：创建新房间，并进入棋盘等待状态。
- 输入有效房间码点击“开始”：加入对应房间。
- 三名玩家按加入顺序入座：魏、蜀、吴。
- 第四名及之后加入者为观战者。
- 三名玩家到齐后自动开始，按规则核心的回合顺序行棋。
- 服务端是权威裁决端，客户端只提交走子意图。
- 在线对局中主动退出会弹出页面内确认层；确认退出后本方认输出局。
- 玩家执蜀时棋盘顺时针旋转 120 度，执吴时顺时针旋转 240 度，方便当前执棋方正向看棋盘。
- 在线对局和玩家对 AI 对局保留行棋动画；AI 自学习保持快速动画和快速响应。

暂不包含：

- 账号体系
- 匹配大厅
- 聊天
- 计时器
- 悔棋
- AI 补位
- 数据库持久化
- 多 Node 实例间房间同步

## 3. 联机协议

协议类型定义在 `src/online/protocol.ts`。

客户端消息：

| 类型 | 用途 |
| --- | --- |
| `createRoom` | 创建房间，携带创建者 `playerId` 和规则选项 |
| `joinRoom` | 加入房间，携带 5 位数字房间码和 `playerId` |
| `leaveRoom` | 离开房间，不判负；主要用于等待状态、观战或普通断线处理 |
| `forfeitRoom` | 主动退出正在进行的在线对局，服务端判定本方认输 |
| `submitMove` | 提交走子意图，仅包含 `pieceId`、`target`、`clientMoveId` |
| `ping` | 心跳预留 |

服务端消息：

| 类型 | 用途 |
| --- | --- |
| `roomJoined` | 加入或创建房间成功，返回当前玩家视角的房间快照 |
| `roomState` | 房间状态广播 |
| `moveAccepted` | 当前客户端提交的走子被接受 |
| `moveRejected` | 当前客户端提交的走子被拒绝，附带原因 |
| `playerList` | 玩家列表更新预留 |
| `error` | 通用错误 |
| `pong` | 心跳响应 |

房间快照 `OnlineRoomSnapshot` 包含：

- `roomCode`
- `phase`: `waiting`、`playing`、`finished`
- `gameState`
- `players`
- `spectators`
- 当前客户端的 `seat`
- 当前客户端的 `role`
- `options`
- `reconnectExpiresAt`

## 4. 服务端行为

服务端实现位于 `server/online-room.ts` 和 `server/online-server.ts`。

### 房间码

- 由服务端生成。
- 固定为 5 位数字。
- 加入房间时会校验格式；非 5 位数字直接拒绝。

### 席位分配

玩家按加入顺序入座：

1. 第一位：魏
2. 第二位：蜀
3. 第三位：吴

三名玩家之后加入者进入观战。

### 房间阶段

| 阶段 | 含义 |
| --- | --- |
| `waiting` | 玩家未满三人，显示等待玩家中 |
| `playing` | 三人到齐，开始对局 |
| `finished` | 出现胜者，对局结束 |

### 权威裁决

服务端复用规则核心：

- `applyMove` 校验和执行合法走子。
- `resignKingdom` 处理主动认输。
- 服务端广播权威 `GameState`。

客户端不能提交完整棋局，只能提交走子意图。

### 断线与主动退出

普通断线：

- WebSocket `close` 会调用 `leaveRoom`。
- 玩家标记为离线。
- 短时重连仍可恢复原席位。

主动退出：

- 前端在在线对局进行中点击“退出游戏”会显示自定义确认层。
- 用户确认后发送 `forfeitRoom`。
- 服务端调用 `resignKingdom`，将该席位对应国家标记为出局。
- 如果只剩一个活跃国家，立即产生胜者。

## 5. 前端交互

前端入口位于 `src/app/web-entry.ts`。

### 首页联机卡片

“联机在线”是一个模式卡片：

- 默认只显示标题和说明。
- 选中卡片后，房间码输入框在卡片右侧显示。
- 输入框 placeholder 为精简提示。
- 房间码只允许输入数字，最多 5 位。

### 开始按钮逻辑

选择“联机在线”后：

- 房间码为空：发送 `createRoom`。
- 房间码为 5 位数字：发送 `joinRoom`。
- 房间码格式错误：仅在卡片内显示提示，不进入棋盘。

### 棋盘视角

在线玩家根据席位旋转棋盘：

| 席位 | 视角旋转 |
| --- | --- |
| 魏 | 0 度 |
| 蜀 | 顺时针 120 度 |
| 吴 | 顺时针 240 度 |

绘制层通过 `viewRotation` 旋转 Canvas 内容，点击命中通过 `hitTestBoardPoint` 反向映射回逻辑点位。

### 行棋动画

本地 AI 对局：

- 仍使用 `commitMove` 和 `playMoveAnimation`。
- 用户能看到棋子移动和吃子过程。

联机对局：

- 服务端广播新 `GameState` 后，前端读取 `moveHistory` 最后一手。
- 若该手尚未播放过，则从旧局面推断移动棋子和被吃棋子并播放动画。
- 动画结束后切到服务端权威状态。

AI 自学习：

- 仍保留极短动画时长。
- 用于提高自学习轮次效率。

### 退出确认

已避免使用 `window.confirm`，改为页面内确认层：

- 适合后续移动端 App / WebView 迁移。
- 点击遮罩、取消按钮或 `Esc` 会关闭确认层。
- 在线对局进行中确认退出会判负。
- 等待状态或观战退出不会判负。

## 6. 本地开发

安装依赖：

```bash
npm install
```

只运行前端：

```bash
npm run dev
```

运行前端 + 联机服务：

```bash
npm run dev:online
```

默认地址：

- 前端：`http://127.0.0.1:5173/`
- 联机 WebSocket/服务端：`http://127.0.0.1:4173/`
- WebSocket 路径：`/ws`

开发环境中，如果前端运行在 `5173`，客户端默认连接 `4173/ws`。

也可以通过环境变量覆盖 WebSocket 地址：

```bash
VITE_ONLINE_WS_URL=ws://127.0.0.1:4173/ws npm run dev
```

## 7. 构建与发布

生产构建：

```bash
npm run build
```

构建内容：

1. `tsc` 类型检查前端源码。
2. `vite build` 输出前端静态文件到 `dist/`。
3. `npm run build:server` 类型检查服务端并输出 `dist/server/online-server.cjs`。

启动生产服务：

```bash
npm start
```

生产服务会：

- 读取 `PORT`，默认 `4173`。
- 服务 `dist/` 下的前端静态文件。
- 在 `/ws` 提供 WebSocket 联机服务。

示例：

```bash
PORT=8080 npm start
```

### 部署注意事项

- 当前房间状态只保存在 Node 进程内存中。
- 进程重启会清空所有房间。
- 当前适合单实例部署。
- 如果部署到多实例平台，需要先引入 Redis、数据库或粘性会话，否则同一个房间可能落到不同进程。
- 反向代理需支持 WebSocket Upgrade。
- HTTPS 页面应使用 `wss://`。

## 9. 常见问题排查

### 创建或加入房间失败

检查：

- 联机服务是否运行在 `4173`。
- 前端是否连接到正确 WebSocket 地址。
- 房间码是否为 5 位数字。
- 浏览器控制台是否有 WebSocket 连接错误。

### 本地前端能打开但联机失败

如果只运行了：

```bash
npm run dev
```

则只有前端，没有联机服务。请改用：

```bash
npm run dev:online
```

或单独运行：

```bash
npm run start:server:dev
```

### 生产环境刷新后房间丢失

当前房间只在内存中，服务进程重启会清空房间。这是第一版设计限制。

### 多实例部署时房间不稳定

当前不支持多实例共享房间。需要：

- 粘性会话，或
- 共享房间存储，或
- 将房间服务拆成单独实时服务。
