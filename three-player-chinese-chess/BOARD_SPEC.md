# 三人中国象棋棋盘规则基础说明

本文档描述当前网页版棋盘的逻辑结构，用于后续实现棋子落子、移动、路径判断、区域判断和界河规则。

## 1. 坐标系统

棋盘所有可落子点都使用 `行字母 + 列数字` 表示，例如 `A5`、`H3`、`O9`。

- 列方向：`1-9`
- 行方向：每个国度 5 行，共 15 行
- 每个国度有 `5 * 9 = 45` 个连接点
- 全棋盘共有 `3 * 45 = 135` 个连接点

行字母与国度对应：

| 国度 | 行范围 | 中心侧行 | 外侧行 | 连接点范围 |
| --- | --- | --- | --- | --- |
| 魏 | `A-E` | `A` | `E` | `A1-E9` |
| 吴 | `F-J` | `F` | `J` | `F1-J9` |
| 蜀 | `K-O` | `K` | `O` | `K1-O9` |

说明：

- `A/F/K` 是靠近棋盘中心三角形的行。
- `E/J/O` 是各自国度外侧行。
- 数字 `5` 是每个国度的中轴列。

## 2. 连接点

所有连接点如下：

| 国度 | 连接点 |
| --- | --- |
| 魏 | `A1-A9`, `B1-B9`, `C1-C9`, `D1-D9`, `E1-E9` |
| 吴 | `F1-F9`, `G1-G9`, `H1-H9`, `I1-I9`, `J1-J9` |
| 蜀 | `K1-K9`, `L1-L9`, `M1-M9`, `N1-N9`, `O1-O9` |

规则引擎可以按以下方式生成点：

```js
const kingdoms = {
  wei: ["A", "B", "C", "D", "E"],
  wu: ["F", "G", "H", "I", "J"],
  shu: ["K", "L", "M", "N", "O"],
};

const cols = [1, 2, 3, 4, 5, 6, 7, 8, 9];
```

## 3. 普通连接线

普通连接线表示棋子可以沿棋盘线移动时使用的基础图边。

每个国度内部都有两类普通连接线。

横向连接线：

- 对每一行 `R`，连接 `R1-R2`, `R2-R3`, ..., `R8-R9`
- 适用于 `A-O` 所有行

纵向连接线：

- 魏：`A1-B1` 到 `D9-E9`
- 吴：`F1-G1` 到 `I9-J9`
- 蜀：`K1-L1` 到 `N9-O9`

等价生成规则：

```js
// 横向
for (const row of allRows) {
  for (let col = 1; col < 9; col += 1) {
    addEdge(`${row}${col}`, `${row}${col + 1}`, "normal");
  }
}

// 纵向
for (const rows of Object.values(kingdoms)) {
  for (let rowIndex = 0; rowIndex < rows.length - 1; rowIndex += 1) {
    for (const col of cols) {
      addEdge(`${rows[rowIndex]}${col}`, `${rows[rowIndex + 1]}${col}`, "normal");
    }
  }
}
```

## 4. 九宫连接线

每个国度的九宫区域为该国度后 3 行、`4-6` 列。

| 国度 | 九宫区域 | 九宫中心 | 斜线连接 |
| --- | --- | --- | --- |
| 魏 | `C4-C6-E6-E4` | `D5` | `C4-D5`, `D5-E6`, `C6-D5`, `D5-E4` |
| 吴 | `H4-H6-J6-J4` | `I5` | `H4-I5`, `I5-J6`, `H6-I5`, `I5-J4` |
| 蜀 | `M4-M6-O6-O4` | `N5` | `M4-N5`, `N5-O6`, `M6-N5`, `N5-O4` |

这些斜线是正式连接线，应加入移动图。

九宫区域线条在画面中加粗，但加粗只影响视觉，不改变连接关系。

## 5. 跨国连接线

当前棋盘有 6 条跨国连接线。

外围正六边形补线：

| 名称 | 连接线 | 说明 |
| --- | --- | --- |
| 魏吴外界线 | `A1-F9` | 魏与吴之间的外侧闭合边 |
| 吴蜀外界线 | `F1-K9` | 吴与蜀之间的外侧闭合边 |
| 蜀魏外界线 | `K1-A9` | 蜀与魏之间的外侧闭合边 |

内部等边三角形：

| 名称 | 连接线 | 说明 |
| --- | --- | --- |
| 魏吴内界线 | `A5-F5` | 中心三角形一边 |
| 吴蜀内界线 | `F5-K5` | 中心三角形一边 |
| 蜀魏内界线 | `K5-A5` | 中心三角形一边 |

这些线也是正式连接线，应加入移动图，类型建议标记为 `inter_kingdom`。

## 5.1 跨界行棋展开线

界河空白区域虽然只画边界线，但行棋规则中应把相邻国度的中心侧行展开为连续网格。车、炮、马、兵等需要按展开后的连续关系计算，而不是必须先停在边界连接点。

魏吴展开线：

| 魏侧 | 吴侧 |
| --- | --- |
| `E1-D1-C1-B1-A1` | `F9-G9-H9-I9-J9` |
| `E2-D2-C2-B2-A2` | `F8-G8-H8-I8-J8` |
| `E3-D3-C3-B3-A3` | `F7-G7-H7-I7-J7` |
| `E4-D4-C4-B4-A4` | `F6-G6-H6-I6-J6` |
| `E5-D5-C5-B5-A5` | `F5-G5-H5-I5-J5` |

吴蜀展开线：

| 吴侧 | 蜀侧 |
| --- | --- |
| `J1-I1-H1-G1-F1` | `K9-L9-M9-N9-O9` |
| `J2-I2-H2-G2-F2` | `K8-L8-M8-N8-O8` |
| `J3-I3-H3-G3-F3` | `K7-L7-M7-N7-O7` |
| `J4-I4-H4-G4-F4` | `K6-L6-M6-N6-O6` |
| `J5-I5-H5-G5-F5` | `K5-L5-M5-N5-O5` |

蜀魏展开线：

| 蜀侧 | 魏侧 |
| --- | --- |
| `O1-N1-M1-L1-K1` | `A9-B9-C9-D9-E9` |
| `O2-N2-M2-L2-K2` | `A8-B8-C8-D8-E8` |
| `O3-N3-M3-L3-K3` | `A7-B7-C7-D7-E7` |
| `O4-N4-M4-L4-K4` | `A6-B6-C6-D6-E6` |
| `O5-N5-M5-L5-K5` | `A5-B5-C5-D5-E5` |

规则说明：

- 车：可沿上述展开线一次走入相邻国度，中途不能越子。
- 炮：可沿上述展开线移动和隔子吃子。
- 马：可在相邻国度展开网格中按“日”字跳，仍受马腿限制。
- 兵：到达中心侧行后，可一步跨过界河进入相邻国度；过界后可继续向前或左右移动，不可后退。

## 6. 界河

界河是三个国度之间的空白区域。界河本身没有额外落子点，棋子只能沿界河边界上的连接线和连接点移动。

| 界河 | 相邻国度 | 外侧边 | 内侧边 | 边界点 |
| --- | --- | --- | --- | --- |
| 赤壁界 | 魏、吴 | `A1-F9` | `A5-F5` | `A1`, `A5`, `F5`, `F9` |
| 荆州界 | 吴、蜀 | `F1-K9` | `F5-K5` | `F1`, `F5`, `K5`, `K9` |
| 岐山界 | 蜀、魏 | `K1-A9` | `K5-A5` | `K1`, `K5`, `A5`, `A9` |

界河文字只用于显示：

- `赤壁` 放在魏吴之间的空白区域，竖排，朝向 `A5-F5`
- `荆州` 放在吴蜀之间的空白区域，竖排，朝向 `F5-K5`
- `岐山` 放在蜀魏之间的空白区域，竖排，朝向 `K5-A5`

## 7. 国度信息

国度用于判断棋子初始归属、阵营区域、宫区和后续特殊规则。

| 国度 | 显示文字 | 点范围 | 宫区 | 中轴 |
| --- | --- | --- | --- | --- |
| 魏 | `魏` | `A1-E9` | `C4-C6-E6-E4` | `A5-E5` |
| 吴 | `吴` | `F1-J9` | `H4-H6-J6-J4` | `F5-J5` |
| 蜀 | `蜀` | `K1-O9` | `M4-M6-O6-O4` | `K5-O5` |

棋子颜色：

| 国度 | 棋子颜色 | 兵卒显示 |
| --- | --- | --- |
| 魏 | 红色 | `兵` |
| 吴 | 深蓝色 | `卒` |
| 蜀 | 深绿色 | `兵` |

行棋顺序：

```text
魏 -> 蜀 -> 吴 -> 魏
```

即魏先行，然后按逆时针顺序行棋。

建议的数据结构：

```js
const kingdomOfRow = {
  A: "wei", B: "wei", C: "wei", D: "wei", E: "wei",
  F: "wu", G: "wu", H: "wu", I: "wu", J: "wu",
  K: "shu", L: "shu", M: "shu", N: "shu", O: "shu",
};

function kingdomOf(pointId) {
  return kingdomOfRow[pointId[0]];
}
```

## 8.2 将军与胜负

当前实现支持两种出局条件：

| 配置 | 行为 |
| --- | --- |
| 主公被将死 | 某国处于将军状态，且无法通过任意一步解除该国被将军状态，则该国出局 |
| 主公被吃掉 | 某国主公被实际吃掉时，该国出局 |

- 每步棋后检测所有国度是否被将军。
- 当前交互会显示棋子自身的全部行棋提示；是否解除将军由玩家自行判断。
- 当只剩一个未出局国家时，该国获胜。
- 出局方剩余棋子的处理由开局设置决定。

出局棋子处理配置：

| 配置 | 行为 | 界面表现 |
| --- | --- | --- |
| 移除 | 出局后该国全部棋子从棋盘移除 | 棋子直接消失 |
| 障碍 | 出局棋子留在棋盘上，只作为障碍物 | 棋子变为浅灰色 |
| 接管 | 吃掉主帅的国家接管出局方剩余棋子 | 两个圆圈颜色变为接管方颜色，文字颜色保持原国度颜色 |

说明：三人棋存在多种胜负变体，包括“将死即胜”“吃将才胜”“淘汰后继续”等。当前实现优先采用更接近三国象棋资料中“需将老将吃到手才算”的规则。

## 9. 移动图构建建议

规则引擎可以把棋盘抽象成无向图：

```js
const graph = {
  nodes: Set<pointId>,
  edges: Array<{
    from: pointId,
    to: pointId,
    type: "normal" | "palace" | "inter_kingdom",
  }>,
};
```

构建顺序建议：

1. 生成 135 个连接点。
2. 添加所有国度内部横向、纵向普通连接线。
3. 添加 12 条九宫斜线，类型为 `palace`。
4. 添加 6 条跨国连接线，类型为 `inter_kingdom`。
5. 记录 3 个界河区域，用于后续实现过界、河界、阵营转换或特殊走法。

当前棋盘没有额外隐藏连接点；所有可落子点都必须是本文档第 2 节列出的连接点。
