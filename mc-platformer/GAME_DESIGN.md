# 《方块跳跃》游戏设计文档

> 版本：与当前仓库代码一致  
> 技术形态：单页 HTML5 Canvas 2D + 原生 JavaScript，无构建链

---

## 1. 作品概述

### 1.1 一句话定位

横版卷轴**平台跳跃**小游戏，美术与气质参考《我的世界》（方块史蒂夫、苦力怕、幻翼、钻石），核心循环为**闯关 → 终点红钻过关 → 五关通关排行**。

### 1.2 目标玩家

休闲玩家、MC 题材爱好者；单局时长数分钟，可重复挑战蓝钻收集与更短用时。

### 1.3 核心乐趣

- **位移技巧**：二段跳、软起步与空中摩擦等手感调参，跨越断层与木板。  
- **风险博弈**：地面苦力怕巡逻、空中幻翼追击；靠近苦力怕可用**砍刀**化解，中远距离用**有限弹药**点杀幻翼。  
- **收集与评价**：蓝钻加分；通关后本地排行榜（蓝钻优先、用时次之）。

---

## 2. 世界观与叙事

叙事极弱，仅通过方块地形与敌对生物建立「MC 地下城/空岛」联想。无剧情文本，关卡标题（草地练习、矿脉石桥等）承担弱引导。

---

## 3. 操作与控制

| 输入 | 行为 |
|------|------|
| **A / D** 或 **← / →** | 水平移动；同时影响史蒂夫朝向 |
| **W** 或 **↑** | 跳跃；空中再按触发**二段跳**（每离地一次） |
| **空格** | 若与苦力怕**近战范围**重叠：挥**砍刀**击杀（不耗弹药）；否则发射**自动瞄向最近幻翼**的飞弹（消耗弹药，场上无幻翼不发射） |
| **死亡遮罩** | 点击「重生」或 **Enter** 等价于确认重生 |
| **重新开始** | 页眉按钮：整局重置（分数、关卡、生命等） |

**设计意图**：将「跳跃」与「射击」拆到不同键位，避免空格同时承担二段跳与开火。

---

## 4. 游戏规则

### 4.1 胜利与流程

- 共 **5 关**。每关存在一颗**红色终点钻石**，玩家碰撞后进入**过关结算**，再进入下一关。  
- 第 5 关红钻拾取后进入**全部通关**结算，写入**本地排行榜**，可「再玩一局」。

### 4.2 失败与重生

- **生命**：初始 6；与苦力怕或幻翼碰撞扣 1，无敌帧内不可再伤。  
- **坠落**：玩家 y 低于画布下边界阈值则**生命清零**，进入死亡遮罩。  
- **生命归零**：进入 **dead** 状态；重生后**保留蓝钻分数与已捡蓝钻状态**，生命补满，回到**本关起点**，幻翼系统整关重置。

### 4.3 得分与排行

- **蓝钻**：每关 **6** 颗，每颗 +1 分，仅作收集与排行依据，非过关必要条件。  
- **计时**：仅在 `play` 状态累计显示为 `分:秒`。  
- **排行榜**：`localStorage` 存最近若干条；排序规则：**蓝钻数降序**，相同则**用时升序**；结算页展示前 10。

### 4.4 弹药

- 每关开始 **5 发**远程弹药；**进关**与**死亡重生**时重置。  
- 近战击杀苦力怕**不消耗**弹药。

## 5. 关卡与内容配置

### 5.1 关卡数据（`LEVELS`）

每关包含：

- **plat**：平台列表，每项为 `[格子 x, 格子 y, 宽度(格), 类型]`，类型为 `grass` / `stone` / `plank`。  
- **spawnX / spawnY**：玩家出生世界坐标（像素）。  
- **redDiamond**：终点红钻参考格坐标，经「贴平台顶」逻辑换算为实际浮动位置。  
- **蓝钻**：每关固定 **6** 颗，沿关卡宽度**均匀分桶**放置，避开出生禁区与苦力怕。  
- **creepersPatrol**：整数数组；**不表示苦力怕只数**，而是与「每关 6 只」循环结合，用于**微调巡逻水平速度**；苦力怕在**脚下整块平台**内左右巡逻。

### 5.2 敌人数量与分布

- **苦力怕**：每关 **6 只**，打死后不刷新；沿关卡宽度均匀分桶、**避开出生禁区**；**同一平台矩形段（台阶）上至多 1 只**（`platformSegmentKey`）。在脚下整块平台内左右巡逻。  
- **幻翼**：每关 **总共生成 6 只**（`phantomsSpawnedTotal`），从 0 只起约 **0.65～1.3 秒**出第一只，再按递增间隔逐只出现；**击杀后不补员**。位置按第 k 只桶位 k=0..5 均匀。  
- **蓝钻**：每关 6 颗，均匀分桶，避开出生区与苦力怕。

## 6. 实体与行为规格

### 6.1 玩家（史蒂夫）

- **碰撞盒**：AABB，与平台做分离轴解算（顶/底/左右最小穿透优先）。  
- **动画**：脚底为变换原点；跑动倾斜、摆臂摆腿、空中轻微拉伸/压缩、头部微动；无敌时半透闪烁。

### 6.2 苦力怕

- 在随机平台上生成，**minX / maxX** 为该平台可行走内边界。  
- 匀速往返，`facing` 与速度符号同步，绘制水平翻转。  
- **同一平台段（同 x/y/w/h 矩形）仅分配一只苦力怕**，避免叠在同一台阶上。

### 6.3 幻翼

- 初始散布在关卡水平范围内；持续朝玩家上半身方向施加加速度，叠加正弦扰动与阻尼、速度上限。  
- y 钳制在画面上方带状区域；超出镜头左右时**环绕**到另一侧，保持压迫感。

### 6.4 飞弹

- 从玩家上半身附近生成，方向指向**当时最近的幻翼中心**；直线运动直至命中或飞出回收范围。

## 7. 状态机（与代码一致）

| 状态 | 说明 |
|------|------|
| `menu` | 开局前遮罩 |
| `play` | 正常游玩，物理与 AI 更新 |
| `dead` | 生命归零，待重生 |
| `intermission` | 过关后待点「下一关」 |
| `win` | 五关通，展示排行榜 |

---

## 8. 视听与反馈

- **画面**：Canvas 自绘像素风；扣血全屏闪红 + 镜头随机抖动。  
- **音频**（`audio.js`）：Web Audio 合成 BGM 与 SFX；需用户手势 `unlock`。含跳跃、射击、幻翼命中、砍苦力怕、拾钻、红钻、受伤、死亡、过关、通关等。

---

## 9. 技术架构与文件职责

| 文件 | 职责 |
|------|------|
| `index.html` | 壳子、HUD、画布、遮罩、脚本引入顺序 |
| `style.css` | 布局与主题色、遮罩与结算表样式 |
| `audio.js` | `window.GameAudio`：BGM + 音效 API |
| `game.js` | 主循环、物理、关卡、绘制、输入、排行榜 |

**主循环**：`requestAnimationFrame`；`play` 时顺序为 **计时 → resolvePlayer → updateCollectibles → 绘制**（世界层 clip，震屏与闪红在 clip 外）。

---

## 10. 已知限制与扩展方向（非承诺）

- 无服务端存档，排行仅本机。  
- 关卡为静态表驱动，可扩展为外部 JSON / 关卡编辑器导出。  
- 可增加手柄支持、键位自定义、无障碍（减少纯红闪）等。
